# 小红书笔记生命周期分析器

这是一个 Windows 本地网页工具，用来分析小红书导出的笔记 CSV 数据。

它可以帮你看到每篇笔记已经发布了多久、观看量和互动表现怎么样，并根据规则给出状态和运营建议。

第一版只做单次 CSV 快照分析，不联网、不上传数据、不需要安装任何软件。

## 如何打开

1. 打开这个文件夹。
2. 找到 `index.html`。
3. 用鼠标双击 `index.html`。
4. 浏览器打开后，就可以使用工具。

## 如何把 Excel 另存为 CSV

如果你从小红书导出的数据是 Excel 文件，可以这样做：

1. 用 Excel 打开数据表。
2. 点击左上角“文件”。
3. 选择“另存为”。
4. 文件类型选择 `CSV UTF-8（逗号分隔）` 或 `CSV（逗号分隔）`。
5. 保存后，会得到一个 `.csv` 文件。

如果 Excel 提示“部分功能可能不兼容 CSV”，这是正常提示，确认保存即可。

## 如何上传 CSV

1. 在页面顶部找到“上传 CSV 文件”。
2. 点击选择文件。
3. 选择你刚才保存的 `.csv` 文件。
4. 页面提示读取成功后，继续填写当前观察时间。

CSV 表头需要包含这些字段：

```text
笔记标题
首次发布时间
体裁
曝光
观看量
封面点击率
点赞
评论
收藏
涨粉
分享
人均观看时长
弹幕
```

## 单份 CSV 分析和多次 CSV 快照对比

这个工具现在有两种用法。

`单份 CSV 分析`：

上传一份小红书 CSV，输入当前观察时间，工具会分析每条笔记当前处于什么状态，比如爆款、普通、疑似尾流等。

`多次 CSV 快照对比`：

上传多个不同时间点导出的小红书 CSV，比如上午 10:00 一份、下午 14:00 一份、晚上 18:00 一份。每个快照都需要填写对应的观察时间。

工具会按观察时间从早到晚自动排序，然后比较相邻快照，比如 10:00 -> 14:00、14:00 -> 18:00。

这些 CSV 最好来自同一个小红书账号，并且导出范围尽量一致。

每个快照观察时间必须不同，后一个时间必须晚于前一个时间。

新增数据的计算方式是：

```text
新增数据 = 后一个快照数据 - 前一个快照数据
```

例如：

```text
前快照观看量 1,000
后快照观看量 1,260
新增观看量 260
```

如果出现负数，通常不是工具出错，可能是：

- 小红书平台统计修正
- 两次 CSV 导出范围不一致
- 多份 CSV 不是同一个账号或不是同一批笔记

多次 CSV 快照对比可以帮助你判断：

- 哪些作品还在增长
- 哪些作品疑似尾流
- 哪些作品突然放量
- 账号整体新增了多少观看、赞藏、收藏

多快照对比第一版会生成每个相邻区间的结果。它适合先看某几个时间段内的新增变化，还不是复杂的长期趋势图。

多快照对比表的筛选顺序是：先按“对比区间筛选”，再按“增长判断筛选”。所以当你切换到某一个具体区间时，“增长判断筛选”里的选项和数量也会跟着变化。

例如全部区间里有 `明显放量（4）`，但当前区间里只有 1 条明显放量，那么切换到这个区间后会显示 `明显放量（1）`。筛选项后面的数字表示当前区间内对应增长判断的数量。

## 如何输入当前观察时间

当前观察时间就是你分析这份数据时的时间。

推荐格式：

```text
2026-05-27 17:30
```

工具会用这个时间减去每篇笔记的“首次发布时间”，计算每篇笔记已经发布了多少小时。

## 如何理解分析结果

页面上方是总览卡片，可以快速看到：

- 总笔记数
- 总曝光
- 总观看量
- 总点赞
- 总收藏
- 总赞藏和
- 平均封面点击率
- 平均赞藏率
- 平均收藏率

顶部总览现在分成两块：

- `核心数据总览`：显示固定核心指标，比如总笔记数、总曝光、总观看量、总点赞、总收藏、总赞藏和。
- `状态分布`：根据当前 CSV 中真实出现的状态自动生成，比如普通、失败、爆款、疑似尾流。

当前 CSV 中没有出现的状态，不会显示在状态分布里。状态分布和状态筛选下拉框使用同一套状态统计和排序规则，数量应该保持一致。

状态分布默认基于全部数据，不受筛选影响。

下方表格会展示每条笔记的原始数据、计算指标、状态和运营建议。

筛选区分成两类：

- `生命周期筛选`：按发布时间阶段筛选，比如 0-2 小时、2-6 小时、24 小时以上。
- `状态筛选`：根据当前 CSV 中实际出现的状态自动生成，比如普通、失败、爆款、疑似尾流。

状态筛选里只会显示当前数据真实出现过的状态，没有出现的状态不会显示。每个状态后面的数字表示当前 CSV 里有多少条作品属于这个状态。

生命周期筛选和状态筛选可以同时使用。例如同时选择“24 小时以上”和“疑似尾流”，表格只会显示同时满足这两个条件的作品。

“当前筛选结果”的合计数据和表格底部合计行，会跟随筛选结果一起变化；“全部数据”合计不受筛选影响。

常见状态含义：

- `爆款潜力`：发布早期观看量和赞藏率都不错，建议继续观察。
- `初始曝光正常`：早期启动正常，不要太早判断失败。
- `冷启动失败`：2-6 小时内观看和赞藏都很低，暂不建议复用。
- `优质低流量`：互动比例不错，但观看量偏低，可以考虑换封面、标题或发布时间二发。
- `爆款`：观看量和赞藏率都比较强，适合重点复盘。
- `疑似尾流`：以前有一定观看量，但当前每小时观看和赞藏偏低。

## 页面悬浮说明

页面中很多指标名、表头、按钮和状态标签都可以悬停查看说明。

如果你不理解某个指标，可以把鼠标停在指标名或旁边的小问号上，比如“赞藏率”“整体点击转化”“疑似尾流”等。

这些 Tooltip 只解释指标或功能的含义，不会改变任何分析结果。

## 一键生成复盘结论

页面底部有一个 `复盘结论生成` 区域。

它会根据当前页面已经计算出的数据，自动生成一段可以复制到飞书、Excel 或运营记录里的复盘文字。

这个功能不调用外部 AI，不联网，也不会上传你的数据。它只是根据页面里已有的统计结果、状态分布和重点作品，用固定规则整理成文字。

目前有两种复盘：

- `生成单份 CSV 复盘结论`：适合查看当前这一次 CSV 的整体状态，比如总观看、总收藏、状态分布、爆款、疑似尾流和下一步建议。
- `生成多快照增长复盘结论`：适合查看一段时间内的新增数据和增长趋势，比如哪些作品明显放量、哪些作品稳定增长、哪些作品疑似尾流或基本停滞。

生成的结论只是辅助判断，最终运营动作仍然要结合账号实际情况、发布时间、选题规划和近期内容节奏来决定。

如果需要保存复盘内容，可以点击 `复制复盘结论`，再粘贴到你的运营记录里。

## 什么是选题修正库

“选题修正库”是工具帮你记住手动改过的“选题/菜品名”。

比如小红书原始笔记标题是：

```text
在家复刻东北锅包肉，真的太有成就感了
```

你手动把“选题/菜品名”改成：

```text
东北锅包肉
```

工具会把这个修正保存到当前浏览器本地。以后你再次导入新的小红书原始 CSV，只要遇到相同的原始笔记标题，就会优先使用你之前修正过的选题。

选题生成优先级是：

1. CSV 里已经有“选题/菜品名”，就直接使用 CSV 里的内容。
2. 如果选题修正库里有相同原始标题，就使用历史修正。
3. 如果没有历史修正，就自动识别。
4. 如果识别不到，就显示“待填写”。

## 选题修正库如何保存和备份

手动修改的选题会保存在当前浏览器的 `localStorage` 里。

请注意：

- 数据只保存在当前浏览器、当前电脑。
- 清理浏览器缓存或网站数据，可能会导致修正库丢失。
- 换电脑或换浏览器后，原来的修正库不会自动跟过去。

建议定期点击：

```text
导出选题修正库
```

这样可以备份成一个 CSV 文件。

如果换电脑、换浏览器，或者清理缓存后想恢复修正库，可以点击：

```text
导入选题修正库
```

选择之前导出的修正库 CSV，工具就会把历史修正重新写回本地浏览器。

如果导入修正库后页面已经有分析结果，可以点击：

```text
重新应用修正库
```

工具会把当前表格里的选题重新匹配一遍历史修正。

## 重要说明

单份 CSV 分析看到的是某一个时间点的数据快照。

所以单份 CSV 里的“疑似尾流”仍然只是辅助判断，不能绝对说明一条笔记已经进入尾流。

多次 CSV 快照对比可以进一步查看相邻时间段里的新增数据，比如：

- 新增观看
- 新增点赞
- 新增收藏
- 新增评论

如果这些新增数据在多个连续区间里持续变低，才更接近真实尾流判断。

## 账号合计数据

“账号合计数据”用来查看当前上传 CSV 里所有笔记加起来的总表现。

它会统计总曝光、总观看量、总点赞、总收藏、总评论、总分享、总涨粉、总赞藏和、总互动量，也会计算整体赞藏率、整体收藏率、整体评论率、整体分享率、整体涨粉率和整体点击转化。

页面里有两组合计：

- `全部数据`：当前上传 CSV 里的所有笔记，不受筛选影响。
- `当前筛选结果`：当前筛选器选中的笔记，比如只看“爆款”或只看“24 小时以上”。

表格底部也有一行“合计”，统计的是当前筛选结果。排序只改变显示顺序，不会改变合计数。

你可以用两次导出的 CSV 合计值，手动对比账号整体新增数据。

例子：

```text
10:00 导出 CSV：总观看量 10,000，总收藏 800
14:00 导出 CSV：总观看量 12,500，总收藏 920
```

那么：

```text
新增观看量 = 12,500 - 10,000 = 2,500
新增收藏 = 920 - 800 = 120
```

合计数据主要用于页面查看，目前不一定写入“导出带选题 CSV”文件。

## AI 实验版说明

这个文件夹是 AI 实验版，不影响外层稳定版。

AI 实验版保留原来的本地规则分析能力，同时新增 DeepSeek AI 复盘层：

- CSV 解析、数据计算、状态判断、增长判断仍然由浏览器本地规则完成。
- DeepSeek 不参与 CSV 解析，不重新计算原始 CSV。
- 前端只把页面已经算好的结构化摘要发给本地后端。
- API Key 不写进前端 `script.js`，只放在本地 `.env` 文件里。
- 如果 DeepSeek 调用失败，页面会显示 HTTP 状态码、后端返回信息和当前模型名；原来的单份 CSV 分析、多快照对比、选题修正库、筛选和合计功能都可以继续使用。

### 小白启动方式

第一次使用：

1. 双击 `启动AI分析器.bat`。
2. 如果还没有 `.env`，窗口会提示你输入 DeepSeek API Key。
3. 输入后脚本会自动保存到本地 `.env`。
4. 后端启动成功后，浏览器会自动打开：

```text
http://127.0.0.1:8787
```

以后使用：

1. 直接双击 `启动AI分析器.bat`。
2. 如果后端已经在运行，脚本不会重复启动服务，只会直接打开浏览器。
3. 如果后端还没运行，脚本会自动启动后端并打开页面。

如果你的 Windows 环境对中文文件名不稳定，也可以双击英文版：

```text
start-ai-lab.bat
```

如果启动没有反应，请先双击：

```text
diagnose-ai-lab.bat
```

它只做检查，不会启动服务。诊断窗口会显示当前目录、Node 版本、npm 版本、`.env` 是否存在、Key 是否已配置、8787 端口是否被占用、`/api/health` 是否可访问。

### 小白关闭方式

正常使用结束后，可以先关闭浏览器页面。

如果想彻底关闭本地后端，请双击：

```text
关闭AI分析器.bat
```

如果你的 Windows 环境对中文文件名不稳定，也可以双击英文版：

```text
stop-ai-lab.bat
```

关闭脚本会先确认 8787 端口上的服务确实是 `xhs-lifecycle-analyzer-ai-lab`，然后才会关闭对应进程，不会误杀其他程序。

如果网页还能打开，说明后端仍在运行。
如果关闭脚本提示已关闭，再访问 `http://127.0.0.1:8787` 应该打不开。

### .env 配置说明

`.env` 会由启动脚本自动创建，最终结构大致是：

```text
DEEPSEEK_API_KEY=your-deepseek-api-key
DEEPSEEK_MODEL=deepseek-v4-pro
DEEPSEEK_API_URL=https://api.deepseek.com/chat/completions
DEEPSEEK_MAX_TOKENS=16000
DEEPSEEK_AUTO_CONTINUE_LIMIT=3
PORT=8787
```

注意：

- 真实 API Key 只保存在你电脑本地的 `.env` 里。
- 不要把真实 API Key 写进 README。
- 不要把真实 API Key 写进前端 JS。
- `.env` 已经被 `.gitignore` 忽略，建议只保存在自己电脑本地。
- 当前启动脚本会固定写入 `DEEPSEEK_MODEL=deepseek-v4-pro`，用户不需要每次输入模型名。
- 当前后端默认使用 `DEEPSEEK_MAX_TOKENS=16000`，用于减少 AI 复盘文字被截断的概率。
- 如果 DeepSeek API 支持更高输出，你可以在 `.env` 里手动改成 `DEEPSEEK_MAX_TOKENS=32000`，但不建议写成无限制。
- `DEEPSEEK_AUTO_CONTINUE_LIMIT=3` 表示如果模型因为长度停止，后端最多自动续写 3 次。
- `max_tokens` 是模型单次输出上限，不是中文字数；输出越长，DeepSeek API token 成本越高。
- 如果 DeepSeek 实际服务要求的模型 ID 后续发生变化，可以修改 `server.js` 里的 `DEFAULT_DEEPSEEK_MODEL`，也可以修改 `.env` 里的 `DEEPSEEK_MODEL` 后重新启动。

### 常见问题

如果 Key 填错：

1. 打开 AI 实验版文件夹里的 `.env`。
2. 修改 `DEEPSEEK_API_KEY`。
3. 保存后重新双击 `启动AI分析器.bat`。

如果页面打不开：

- 先看 PowerShell 窗口里的错误提示。
- 检查电脑是否安装了 Node.js。
- 检查 8787 端口是否被其他程序占用。

如果 AI 分析失败：

- 检查 API Key 是否正确。
- 检查 `DEEPSEEK_MODEL` 是否为 `deepseek-v4-pro`。
- 检查 `DEEPSEEK_MAX_TOKENS` 是否为 `16000`。如果模型不支持这么高，后端会自动降级到 `8000` 重试。
- 如果 AI 结论仍然被截断，工具会自动续写；如果续写 3 次后仍不完整，可以减少输入快照数量，或在模型支持的范围内继续提高 `DEEPSEEK_MAX_TOKENS`。
- 检查网络是否能访问 DeepSeek。
- 检查 DeepSeek 账号余额是否正常。
- 原来的 CSV 分析、多快照对比、选题修正库、筛选和合计功能仍然可以正常使用。

如果你直接双击 `index.html`，本地规则分析仍然可以用；但 AI 复盘需要本地后端已经启动。

### 小白排错

如果双击启动没反应：

1. 先双击 `diagnose-ai-lab.bat`。
2. 把诊断窗口截图保存下来。
3. 再尝试双击英文启动文件 `start-ai-lab.bat`。

如果诊断里提示 Node 不存在：

- 说明电脑没有安装 Node.js，或者 Node 没有加入 PATH。
- 安装 Node.js 后，重新打开 `diagnose-ai-lab.bat` 检查。

如果诊断里提示 8787 端口已占用：

- 如果 `/api/health` 可访问，并且 `health.service` 是 `xhs-lifecycle-analyzer-ai-lab`，说明后端已经在运行，直接打开浏览器即可。
- 如果 `/api/health` 不可访问，说明 8787 可能被其他程序占用。先关闭占用端口的程序，再重新启动。

如果诊断里提示 Key 未配置：

- 双击 `启动AI分析器.bat` 或 `start-ai-lab.bat`，按提示输入 DeepSeek API Key。
- 输入后会自动写入 `.env`，以后不用重复输入。

如果页面能打开，但 AI 分析失败：

- 先确认 `.env` 里的 `DEEPSEEK_API_KEY` 是否正确。
- 再确认 `.env` 里的 `DEEPSEEK_MODEL` 是否为 `deepseek-v4-pro`。
- 再确认网络是否能访问 `https://api.deepseek.com`。
- 再确认 DeepSeek 账号余额是否正常。
- 可以在浏览器打开 `http://127.0.0.1:8787/api/deepseek-test`，查看后端返回的 `status`、`message` 和 `model`。
- 这种情况不会影响原来的 CSV 上传、单份分析、多快照对比、选题修正库、筛选和合计功能。

### 如何使用 AI 复盘

1. 先完成单份 CSV 分析，或者完成多次 CSV 快照对比。
2. 点击 `生成 DeepSeek AI 复盘结论`。
3. 页面会把结构化摘要发给本地后端。
4. 后端读取 `.env` 里的 Key，调用 DeepSeek。
5. DeepSeek 返回的复盘文字会显示在复盘文本框里。

复盘区会显示：

- `AI 模型`：从后端 `/api/health` 读取，例如 `deepseek-v4-pro`。
- `AI 分析依据`：根据当前页面数据自动显示是只基于单份 CSV、只基于多快照，还是两者综合。

如果 DeepSeek 返回的 `finish_reason` 表示长度截断，页面会在 AI 结果上方显示提示：

```text
AI 结论可能仍未完整，建议减少输入快照数量或继续提高 max_tokens。
```

如果模型不支持 `.env` 里设置的 `max_tokens`，后端会自动降级到 `8000` 重试，并在页面上方提示：

```text
当前模型不支持设置的 max_tokens，已自动降级为 8000 后重试。
```

页面还会显示实际使用的 `max_tokens`、是否自动续写、续写次数、最终 `finish_reason` 和是否可能被截断。
生成结果后，复盘文本框会自动滚动到顶部，方便从开头阅读。复制按钮会复制完整文本，不只复制当前可见部分。

发送给 AI 的摘要包括：

- 单份 CSV 总览数据
- 状态分布
- 账号合计数据
- 多快照总新增数据
- 各区间新增数据
- 明显放量作品
- 稳定增长作品
- 疑似尾流作品
- 基本停滞作品
- 新增观看最高作品
- 新增赞藏最高作品
- 新增收藏最高作品

不会把完整 CSV 原始内容整份发给模型。

## 从文件名自动识别观察时间

为了减少每次手动输入观察时间，AI 实验版会在你选择 CSV 后，优先从文件名里识别观察时间。

推荐命名格式：

```text
0610 19点45.csv
0606 9点30.csv
0527 12点.csv
0527 12点00.csv
```

识别规则很简单：

- 文件名前 4 位数字表示月日，比如 `0610` 表示 6 月 10 日。
- 后面的 `19点45` 表示 19:45。
- 如果只有 `12点`，分钟默认是 `00`。
- 文件名里没有年份时，默认使用浏览器当前年份。
- 页面会统一填成 `YYYY-MM-DD HH:mm`，例如 `2026-06-10 19:45`。

不推荐命名：

```text
小红书数据.csv
随便命名.csv
```

如果文件名无法识别，工具会尝试使用文件修改时间作为观察时间，并提示你：

```text
来源：文件修改时间，请确认是否准确
```

这个时间不一定等于小红书导出时间，所以建议你确认一下；如果不准确，可以直接在观察时间输入框里手动修改。

多快照对比时，每个快照选择 CSV 后也会自动识别自己的观察时间。工具仍然会按观察时间从早到晚排序后再做相邻快照对比。

## 数据报表看板

页面新增了 `数据报表看板` 区域，用来把已经算好的单份 CSV 分析结果和多快照对比结果做成可视化报表。

它不会重新解析 CSV，也不会修改状态判断规则，只是把页面已有数据换成更直观的图表展示。

主要包括：

- `多快照总新增趋势`：按对比区间展示新增观看量、新增赞藏和、新增收藏的变化。
- `新增观看 Top10 作品`：展示当前区间新增观看最多的作品。
- `新增赞藏和 Top10 作品`：展示当前区间新增点赞和收藏反馈最强的作品。
- `增长判断分布`：展示当前多快照区间里的明显放量、稳定增长、疑似尾流、基本停滞等数量。
- `当前笔记状态分布`：展示单份 CSV 中不同笔记状态的数量。

报表顶部可以切换：

- `全部报表`
- `单份 CSV 报表`
- `多快照增长报表`

多快照相关图表会跟随页面里现有的 `对比区间筛选` 变化。比如你切换到某一个具体区间，新增观看 Top10、新增赞藏和 Top10、增长判断分布都会只看这个区间的数据。

图表只是辅助观察，最终判断仍然建议结合表格明细、复盘结论和你的运营经验一起看。

## 视觉模式说明

AI 实验版的面试展示稳定版目前保留两档 `视觉效果` 开关：

- `极简`：关闭背景动画、漂浮贴片、扫描光、卡片入场动画和复杂 hover，只保留静态高级样式，性能最稳。
- `标准`：默认模式，保留简洁渐变、基础圆角、浅阴影和轻微 hover，不启用持续背景动画，适合面试展示和日常长时间复盘。

两种模式只影响页面显示效果，不会改变 CSV 解析、状态判断、多快照对比、选题修正库、DeepSeek AI 复盘或 token 用量统计。

如果你觉得页面滚动卡顿，可以切到 `极简`。系统也会尊重电脑的“减少动态效果”设置，自动关闭持续动画和扫描光。

### 贴纸装饰系统

为了保证面试展示稳定，页面里的按钮贴纸、小角标、浮动 CSV、火箭、便签、扫描光等复杂装饰已暂时隐藏。

按钮本体仍然是原来的 HTML 按钮，不会改变上传、分析、导出、AI 复盘等功能。

后续如果需要恢复展示型视觉，可以再单独开启装饰层；当前版本优先保证数字、按钮、表格和图表清楚可读。

### 空标题、匹配风险和图表过滤

小红书导出的部分笔记可能没有原始标题。没有标题时，工具无法只靠标题稳定识别同一篇作品，所以会优先使用“空标题 + 首次发布时间”作为长期 key：

```text
EMPTY_TITLE__首次发布时间
```

例如 `EMPTY_TITLE__2026-04-24 16:14:53`。这样同一篇无标题作品在不同日期导出的 CSV 中，只要首次发布时间一致，就能复用你以前手动修正过的选题。

如果极端情况下，多条空标题作品的首次发布时间也完全相同，工具才会追加行号做兜底，避免它们互相覆盖。旧版本中带行号的空标题修正，如果能解析出首次发布时间，会自动迁移到新的长期 key；如果旧缓存里没有首次发布时间，无法自动迁移，重新手动修正一次后就会按新规则长期生效。

多快照对比里，如果作品原始标题为空，工具也会优先用 `EMPTY_TITLE__首次发布时间` 匹配同一篇作品，并在表格里标记 `匹配风险`。这个提示是提醒你标题为空，需要人工确认选题；如果首次发布时间唯一，它已经比单纯依赖行号更稳定。

报表看板默认会排除这些异常记录：

- 快照缺失
- 新增笔记
- 匹配风险
- 原始标题为空且选题仍为“待填写”的记录

这样做是为了避免 Top10 图表被“待填写”或不稳定匹配记录刷屏。需要查看全部记录时，可以打开 `包含异常/未命名记录`，或者直接看下方表格明细。

Top10 有两种统计方式：

- `按作品汇总`：同一作品在多个对比区间里的新增观看、新增赞藏和会合并，只出现一次，适合看真正的重点作品。
- `按区间记录`：保留每个区间的单条记录，同一作品可能在多个区间重复出现，适合看某个时间段的放量情况。

多快照趋势图拆成了两张：

- `多快照新增观看趋势`：只看新增观看量，观察流量速度。
- `多快照互动质量趋势`：看新增赞藏和、新增收藏、新增点赞，避免被观看量的大数值压扁。
