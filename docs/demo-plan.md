# 面试演示计划

## 1. 演示目标

把项目讲成一个真实内容运营场景下的数据分析工具，而不是单纯网页页面。

推荐一句话介绍：

> 这是一个本地运行的小红书内容运营数据分析工具，可以上传后台导出的 CSV，判断笔记生命周期状态，做多快照增长对比，并用 AI 辅助生成运营复盘结论。

## 2. 演示前准备

建议准备：

- 浏览器打开 AI 实验版页面。
- 使用 `标准` 视觉模式。
- 准备 `demo/mock-xhs-notes.csv`。
- 如果要演示 AI，提前配置本地 `.env`。
- 如果不想暴露 API Key，不要打开 `.env` 文件。
- 关闭浏览器书签栏或其他可能泄露隐私的区域。

## 3. 本地演示流程

### 第一步：介绍项目背景

说明小红书后台 CSV 能提供数据，但运营复盘仍然需要人工计算和判断。

可以强调：

- 原始 CSV 不直观。
- 内容运营需要判断生命周期。
- 多次快照才能更接近判断真实增长和疑似尾流。

### 第二步：上传单份 CSV

使用：

```text
demo/mock-xhs-notes.csv
```

演示点：

- 自动读取 CSV。
- 自动解析发布时间。
- 自动计算观看、曝光、点赞、收藏、赞藏率等指标。
- 自动生成状态和运营建议。

### 第三步：讲核心数据总览

重点讲：

- 总曝光。
- 总观看量。
- 总点赞。
- 总收藏。
- 总赞藏和。
- 平均封面点击率。
- 平均赞藏率。
- 状态分布。

说明这些指标对应运营中的“规模”和“质量”。

### 第四步：展示明细表格

演示：

- 表格排序。
- 生命周期筛选。
- 状态筛选。
- 选题/菜品名列。
- 运营建议列。

可以说明工具不只给结果，还保留明细，方便人工判断。

### 第五步：展示选题修正库

演示：

- 修改一个选题/菜品名。
- 说明它会保存到浏览器 localStorage。
- 下次导入相同标题时复用。
- 可以导出修正库备份。

### 第六步：展示多快照对比

如果有多份 mock 或脱敏快照，可以演示：

- 上传多个时间点 CSV。
- 工具按时间排序。
- 计算相邻快照新增观看、新增点赞、新增收藏、新增赞藏。
- 判断明显放量、稳定增长、疑似尾流、基本停滞。

如果暂时只有单份 mock CSV，可以说明这是已有功能，真实演示时需要多份不同时间点 CSV。

### 第七步：展示图表看板

重点看：

- 当前笔记状态分布。
- 多快照新增趋势。
- 新增观看 Top10。
- 新增赞藏和 Top10。
- 增长判断分布。

说明图表是辅助观察，最终仍然可以回到表格明细核对。

### 第八步：展示 AI 复盘

如果已配置 DeepSeek API Key：

1. 点击 `生成 DeepSeek AI 复盘结论`。
2. 展示 AI 模型、分析依据、token 用量。
3. 说明 AI 只读取结构化摘要，不参与 CSV 解析和规则判断。

如果不想调用真实 API：

- 可以只展示本地规则复盘按钮。
- 或说明后续计划增加 AI mock 演示模式。

## 4. Mock 数据演示流程

当前仓库提供：

```text
demo/mock-xhs-notes.csv
```

它适合展示：

- 单份 CSV 上传。
- 数据总览。
- 状态判断。
- 表格明细。
- 选题识别。
- 本地规则复盘。

它不包含真实账号数据，也不包含 API Key。

## 5. AI 功能演示注意事项

- 不要展示 `.env` 文件。
- 不要把 API Key 输入过程录屏。
- 不要把真实 Key 写入 README、截图或前端代码。
- AI 失败时，说明原始数据分析功能仍然可用。
- 强调 AI 是“复盘辅助层”，不是基础计算层。

## 6. 推荐讲法

可以按这个结构讲：

1. 我从真实内容运营复盘问题出发。
2. 先做了本地 CSV 分析器。
3. 再解决真实 CSV 兼容、中文时间、字段清洗。
4. 然后加入生命周期判断。
5. 后续加入多快照增长对比。
6. 再做选题修正库，沉淀人工判断。
7. 最后接入 AI，把结构化数据转成复盘文字。
8. 项目展示了数据处理、产品迭代、安全意识和 AI 辅助能力。

## 7. 在线演示建议

当前 AI 实验版不建议直接公开部署真实后端，因为它依赖 DeepSeek API Key。

如果未来要在线演示：

- 静态稳定版可以考虑 GitHub Pages。
- AI 功能建议先做 mock 模式。
- 不要把真实 API Key 放到前端。
- 不要让公网直接调用你的真实 DeepSeek Key。

## 8. 演示收尾

最后可以强调：

- 这是一个本地工具，保护数据隐私。
- 真实运营 CSV 不建议上传到 GitHub。
- AI 只辅助总结，核心判断仍由本地规则和数据明细支撑。
- 后续可以继续做规则配置、完整 mock 演示和桌面 App。
