# 小红书笔记生命周期分析器 - 迭代记录

## v1.0 已实现功能

当前版本已经可以正常读取真实小红书 CSV，并在本地浏览器里完成基础分析。

已实现：

- 双击 `index.html` 就能打开工具。
- 不需要联网。
- 不需要安装依赖。
- 不需要后端和数据库。
- 支持上传小红书导出的 CSV 文件。
- 支持手动输入“当前观察时间”。
- 可以自动计算每篇笔记已经发布了多久。
- 可以自动计算赞藏和、互动量、赞藏率、收藏率、评论率、分享率、涨粉率。
- 可以自动计算每小时观看和每小时赞藏。
- 可以根据规则给每条笔记打状态标签。
- 可以给每条笔记生成运营建议。
- 页面顶部有总览数据。
- 下方有完整分析表格。
- 表格支持排序。
- 表格支持筛选。
- 表头可以固定在顶部。
- 前两列“笔记标题”和“首次发布时间”可以固定，方便横向查看。
- 发布时间会简化显示成类似 `2026-05-27 10:30`。
- 已发布时长会显示成更容易理解的格式，比如 `35分钟`、`6.3小时`、`8天1小时`。
- 鼠标悬停在时间上，可以看到原始完整时间。
- 已去掉“体裁”展示列，节省表格空间。

## v1.0 已做的兼容处理

为了适配真实小红书导出的 CSV，当前版本已经做了这些兼容：

- 不再默认第一行就是字段表头。
- 可以自动跳过 CSV 前面的说明文字。
- 会在前 10 行里寻找真正的字段表头。
- 支持 UTF-8 BOM。
- 支持 Windows 换行。
- 支持空行。
- 支持字段前后有空格。
- 支持字段被双引号包裹。
- 支持字段里带英文逗号。
- 支持英文逗号、Tab、分号作为 CSV 分隔符。
- 支持部分字段别名，比如“标题”“作品名称”“小眼睛”“阅读量”等。
- 支持小红书中文时间格式，比如 `2026年05月27日10时30分11秒`。
- `体裁` 字段不再是必需字段，有没有都不影响分析。

## 当前限制

v1.0 主要还是一个“单次快照分析工具”。

当前限制：

- 只能分析一份 CSV。
- 不能保存历史记录。
- 刷新页面后，上传的数据和分析结果会消失。
- 不能直接读取 Excel 文件，需要先把 Excel 另存为 CSV。
- 不能导出分析结果。
- 不能对比两次 CSV 之间新增了多少观看、点赞、收藏、评论。
- 不能真正判断尾流，只能判断“疑似尾流”。
- 判断规则是固定的，暂时不能在页面上自己调整阈值。
- 暂时没有图表。
- 暂时不能按发布时间段、内容类型、标题关键词做更细的复盘。

## 待优化清单

后续可以优先考虑这些优化：

1. 支持多次 CSV 快照对比

   这是第二版最重要的功能。通过对比两份或多份 CSV，可以看到每条笔记新增了多少观看、点赞、收藏、评论，从而更准确判断真实尾流。

2. 支持导出分析结果

   把分析后的表格导出成 CSV，方便继续用 Excel 复盘或留档。

3. 支持保存历史记录

   让工具可以记住之前上传过的数据，方便后续继续分析。

4. 支持自定义判断规则

   允许用户自己调整爆款、优质、冷启动失败、尾流等判断标准。

5. 增加图表

   比如观看量分布、赞藏率分布、发布时间表现、不同状态数量统计。

6. 增加单条笔记详情

   点击一条笔记后，显示更详细的解释：为什么是这个状态，下一步该怎么处理。

7. 优化真实尾流判断

   第二版可以通过“新增观看”“新增赞藏”“新增评论”的变化，判断一条笔记是不是已经真的进入尾流。

8. 增加复盘维度

   后续可以按发布时间、标题关键词、选题方向、封面点击率区间等维度做统计，帮助找到更容易出数据的内容类型。
